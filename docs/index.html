About Crit

Crit (Critical Hit) is a statically typed, educational programming language with a clean C-like syntax that compiles directly to JavaScript. Its name comes from the tabletop RPG concept of rolling a natural 20 — a perfect strike. In Crit, the compiler is your dungeon master: it catches your mistakes before they ever reach runtime. Every type error, undeclared variable, and argument mismatch is flagged at compile time so your program lands clean every time.

The compiler is implemented in JavaScript using the Ohm.js parsing framework and follows a classic three-phase design: parsing, semantic analysis, and code generation, with an optimization pass that performs constant folding. Crit is intentionally minimal — its purpose is to be readable, teachable, and correct.

Whether you are learning about compilers, exploring language design, or just want a language where the rules are enforced by the toolchain rather than by convention, Crit is built for you. Roll for initiative.

Features

Static Typing

Types are checked at compile time. Annotate variables and function signatures with : int, : bool, etc.

Type Inference

Use let x = 10; and the compiler infers the type — no annotation required.

Structs

Define custom data structures with typed fields. Struct instances are compiled to ES6 classes.

Functions

First-class functions with typed parameters and return types. Recursive calls are fully supported.

Control Flow

if/else, while, break, and return — all with compile-time validation.

Constant Folding

The optimizer evaluates constant expressions at compile time. 2 + 3 * 4 becomes 14 in the output.

Semantic Checks

Undeclared variables, duplicate declarations, break-outside-loop, type mismatches, and arg-count errors all throw at compile time.

Compiles to JS

Output is clean, readable JavaScript. Run the result directly with Node.js — no runtime library needed.

Example Programs

Fibonacci

fibonacci.crit
fn fib(n: int) -> int {
  if n <= 1 {
    return n;
  }
  return fib(n - 1) + fib(n - 2);
}

print(fib(10));   // 55

Structs

structs.crit
struct Point {
  x: int
  y: int
}

fn add_points(p1: Point, p2: Point) -> Point {
  return Point(p1.x + p2.x, p1.y + p2.y);
}

let p1 = Point(1, 2);
let p2 = Point(3, 4);
let p3 = add_points(p1, p2);
print(p3.x);  // 4
print(p3.y);  // 6

While Loop with Break

while_break.crit
let i: int = 0;
while i < 10 {
  if i == 5 {
    break;
  }
  print(i);   // prints 0 1 2 3 4
  i = i + 1;
}

Boolean Logic

boolean.crit
fn is_teen(age: int) -> bool {
  return age >= 13 && age <= 19;
}

fn can_vote(age: int, citizen: bool) -> bool {
  return age >= 18 && citizen;
}

let teen: bool = is_teen(15);
let vote: bool = can_vote(20, true);
print(teen);  // true
print(vote);  // true

Function Composition

composition.crit
fn double(n: int) -> int { return n * 2; }
fn increment(n: int) -> int { return n + 1; }
fn square(n: int) -> int { return n * n; }

// square(increment(double(3))) = square(7) = 49
let result: int = square(increment(double(3)));
print(result);  // 49

Arithmetic & Constant Folding

arithmetic.crit
fn square(n: int) -> int { return n * n; }
fn average(a: int, b: int) -> int { return (a + b) / 2; }

print(square(7));          // 49
print(average(10, 20));    // 15

// Optimizer folds this at compile time:
let x: int = 3 + 4 * 2;   // becomes: let x = 11
let y: int = (3 + 4) * 2; // becomes: let y = 14

Block VarDecl = "let" id ":" Type "=" Exp ";" -- with type | "let" id "=" Exp ";" -- inferred Type = "int" | "string" | "bool" | "void" | id Stmt = "return" Exp?
